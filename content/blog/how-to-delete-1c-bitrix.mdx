Иногда возникает необходимость удалить сайт на&nbsp;битриксе и&nbsp;забыть все, что с&nbsp;ним связано.
Но&nbsp;это не&nbsp;такая уж&nbsp;и&nbsp;простая задача, т.к. существуют зависимости, ссылающиеся на&nbsp;сайт, которые нужно удалить. Как это сделать? Сейчас покажу.

Ниже краткая инструкция для удаления сайта 1С-Битрикс.
В&nbsp;моем случае было установлено решение Аспро Next. Порядок удаления может отличаться.

## Ошибки при удалении сайта на CMS 1C-Bitrix
### CRubric::OnBeforeLangDelete (subscribe)

  ```tsx
  CRubric::OnBeforeLangDelete (subscribe)
  ```

Это означает, что нужно удалить списки рассылок.
Для решения проблемы отправляемся в ```Сервисы → Рассылки → Список рассылок``` и&nbsp;удаляем лишние рассылки, привязанные к&nbsp;этому сайту.

### CForumNew::OnBeforeLangDelete (forum)

  ```tsx
  CForumNew::OnBeforeLangDelete (forum)
  ```

Это означает, что нужно отправиться в ```Сервисы → Форумы``` и&nbsp;удалить ненужный форум. Скорее всего это форум &laquo;Отзывы о&nbsp;товарах&raquo;.

После удаления форума возвращаемся в ```Настройки → Сайты → Список сайтов``` и&nbsp;спокойно удаляем сайт.

### CSalePersonType::OnBeforeLangDelete (sale)

  ```tsx
  CSalePersonType::OnBeforeLangDelete (sale)
  ```

Для того чтобы победить ошибку &laquo;Ошибка при удалении в&nbsp;CSalePersonType::OnBeforeLangDelete (sale)&raquo; нужно проделать следующее:
Открыть раздел ```Магазин → Настройки → Типы плательщиков``` и&nbsp;удалить там типы плательщиков, привязанные к&nbsp;тому сайту который мы&nbsp;удаляем. Перед удалением, возможно, потребуется удаление заказов, связанных с&nbsp;этим плательщиком.

### CIBlock::OnBeforeLangDelete (iblock)
  ```tsx
  CIBlock::OnBeforeLangDelete (iblock)
  ```

Надо удалить несколько инфоблоков: Найдены инфоблоки связанные с&nbsp;сайтом. Их&nbsp;идентификаторы: 11, 20, 21, 22, 23, 24. . Идентификаторы в&nbsp;каждом случае свои, но&nbsp;метод решения одинаковый.
Идем в ```Контент → Инфоблоки → Типы инфоблоков``` и&nbsp;удаляем инфоблоки с&nbsp;указанными&nbsp;id. Скорее-всего нужные нам &laquo;жертвы&raquo; раскиданы по&nbsp;разным типам инфоблоков и&nbsp;проверить придется все.
Если видим ошибку при удалении каталога, то&nbsp;заходим в&nbsp;настройки инфоблока и&nbsp;снимаем галочку &laquo;инфоблок имеет торговые предложения&raquo;

Спокойно удаляем сайт.
